# 스킬 보안 스캔 체크리스트

## 위험 패턴 (자동 차단)

| 패턴 | 위험도 | 설명 |
|------|--------|------|
| `API_KEY`, `SECRET`, `TOKEN`, `PASSWORD` | 높음 | 하드코딩된 시크릿 |
| `rm -rf /`, `rm -rf ~`, `rm -rf *` | 높음 | 파괴적 삭제 |
| `DROP TABLE`, `DROP DATABASE`, `TRUNCATE` | 높음 | DB 파괴 |
| `git push --force`, `git reset --hard` | 중간 | 돌이킬 수 없는 git 명령 |
| `eval()`, `exec()`, `system()`, `spawn()` | 중간 | 동적 코드 실행 |
| `chmod 777`, `chmod a+rwx` | 중간 | 과도한 권한 |
| `curl \| sh`, `wget \| bash` | 높음 | 원격 코드 실행 |
| `sudo`, `su -` | 중간 | 권한 상승 |

## 경고 패턴 (사용자 확인 필요)

| 패턴 | 설명 |
|------|------|
| 외부 URL | 신뢰 가능한 도메인인지 확인 |
| 하드코딩된 경로 (`/Users/...`) | `~` 또는 `$HOME`으로 대체 |
| 특정 포트 번호 | 환경별 차이 가능성 |
| 특정 버전 번호 | 업데이트 필요성 |

## 안전한 패턴 (권장)

- `$HOME` 또는 `~` 사용
- 환경 변수로 설정값 분리
- `--dry-run` 옵션 먼저 실행
- 작업 전 `git stash` 또는 백업
- 타임아웃 설정
