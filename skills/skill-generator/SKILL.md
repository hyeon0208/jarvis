# Skill Generator — 자율 스킬 생성

## 역할

복잡한 작업을 수행한 후, 해당 작업을 재현 가능한 스킬로 변환하여 저장합니다.
이를 통해 Jarvis는 경험으로부터 학습하고 성장합니다.

## 실행 절차

이 스킬이 호출되면 다음 단계를 **순서대로** 수행하세요:

### Step 1: 세션 분석

현재 세션에서 수행한 작업을 분석합니다:

1. **세션 상태 파일 확인**: `~/.jarvis/sessions/` 아래에서 현재 세션 파일을 읽으세요
2. **대화 내역 분석**: 이번 세션에서 수행한 전체 작업을 회고하세요
   - 어떤 **문제**를 해결했는가
   - 어떤 **도구**를 사용했는가 (빈도순)
   - 어떤 **파일**을 읽고/수정했는가
   - 어떤 **패턴**이 반복되었는가
   - 핵심 **의사결정**은 무엇이었는가
   - 어떤 **실수**를 했고 어떻게 수정했는가

3. **스킬 후보 도출**: 분석 결과를 바탕으로 스킬화할 가치가 있는 작업을 식별합니다
   - 재사용 가능성이 높은가?
   - 절차가 3단계 이상인가?
   - 다른 프로젝트에서도 적용 가능한가?

### Step 2: 사용자 확인

스킬 후보를 사용자에게 제시하고 확인을 받습니다:
- 스킬 이름 (kebab-case)
- 한 줄 설명
- 주요 단계 요약
- 저장 위치: `~/.claude/skills/{skill-name}/SKILL.md`

**사용자 승인 없이 파일을 생성하지 마세요.**

### Step 3: SKILL.md 생성

승인 후, 다음 형식으로 SKILL.md를 작성합니다:

```markdown
---
name: {스킬명}
description: "{작업 설명 — 언제 이 스킬을 트리거해야 하는지 포함}"
---

# {스킬명}

## Overview
{이 스킬이 해결하는 문제, 적용 시점, 전제 조건}

## Prerequisites
{필요한 도구, 환경, 의존성}

## Steps

### Step 1: {단계명}
{구체적인 수행 절차}
{사용할 도구, 명령, 패턴}

### Step 2: {단계명}
...

## Pitfalls
{이 작업에서 흔히 발생하는 실수와 해결법}

## References
{참고할 문서, 도구, 관련 스킬}
```

**작성 규칙:**
- 절차는 **구체적이고 실행 가능**하게 작성 (모호한 "적절히 처리" 금지)
- 도구 사용법은 **실제 명령/코드**를 포함
- Pitfalls 섹션에 **이번 세션에서 겪은 실수**를 기록
- 시크릿, 토큰, 개인정보를 **절대 포함하지 않기**

### Step 4: 보안 스캔

생성된 SKILL.md를 저장하기 전에 다음을 검사합니다:

```
위험 패턴 체크리스트:
□ API 키, 토큰, 비밀번호가 포함되어 있지 않은가
□ rm -rf, DROP TABLE, git push --force 등 파괴적 명령이 없는가
□ eval(), exec(), system() 등 동적 실행이 없는가
□ 외부 URL이 신뢰할 수 있는 도메인인가
□ 파일 경로에 사용자 홈 디렉토리가 하드코딩되지 않았는가
  (대신 ~ 또는 $HOME 사용)
```

위험 패턴이 발견되면 사용자에게 경고하고 수정을 제안합니다.

### Step 5: 저장 및 인덱싱

보안 스캔 통과 후:

1. **디렉토리 생성**: `~/.claude/skills/{skill-name}/`
2. **SKILL.md 저장**: 위 디렉토리에 SKILL.md 작성
3. **보조 파일 생성** (필요 시):
   - `references/` — 참고 문서, 체크리스트
   - `scripts/` — 자동화 스크립트
   - `assets/` — 템플릿 파일
4. **스킬 인덱스 업데이트**: `node ~/jarvis/hooks/scan-skills.js` 실행
5. **메모리 동기화**: `jarvis_sync_skills` MCP 도구 호출
6. **사용자에게 결과 보고**

### Step 6: 결과 보고

생성된 스킬 정보를 요약합니다:
- 스킬 이름 및 경로
- 포함된 단계 수
- 보안 스캔 결과
- 관련 기존 스킬 (있다면)

## 서브 커맨드

### `/skill-generator` (기본)
현재 세션을 분석하여 스킬을 생성합니다.

### `/skill-generator list`
`jarvis_memory_recall` MCP 도구로 등록된 스킬 목록을 조회합니다:
```
jarvis_memory_list (type: "procedural")
```

### `/skill-generator edit {스킬명}`
기존 스킬의 SKILL.md를 읽고 수정 제안을 합니다:
1. `~/.claude/skills/{스킬명}/SKILL.md` 읽기
2. 개선점 분석
3. 사용자 승인 후 수정

## MCP 도구 의존성
- `jarvis_sync_skills` — 스킬 인덱스 동기화
- `jarvis_memory_recall` — 관련 스킬 검색
- `jarvis_memory_list` — 스킬 목록 조회
